INSTRUCCIONES PARA TERMINAR LA CONFIGURACIÓN DE KAFKA Y WEBSOCKETS
========================================================================

Hola! He configurado todo lo necesario de Kafka y WebSockets para que tu aplicación pueda enviar y recibir mensajes de un chat entre el Repartidor y el Cliente en tiempo real.

Aquí tienes el resumen de lo que ya está implementado en el backend:

1. KAFKA:
   - Dependencias añadidas en el pom.xml (spring-kafka).
   - Configuración en application.properties y KafkaConfig usando StringSerializer para evitar problemas con clases obsoletas de Json.
   - KafkaConfig.java: Define el Topic 'chat-mensajes' de forma automática y configura los beans explícitamente.
   - MensajeProducerService.java: Se encarga de publicar el mensaje en el topic de Kafka como String cada vez que se llama.
   - docker-compose.yml: Creado en la raíz del proyecto usando imágenes oficiales estables de Confluent (confluentinc/cp-zookeeper:7.5.0 y confluentinc/cp-kafka:7.5.0).

2. WEBSOCKETS:
   - Dependencias añadidas en el pom.xml (spring-boot-starter-websocket).
   - WebSocketConfig.java: Configurado el endpoint STOMP '/ws-chat' (con SockJS como fallback) y el broker de mensajes '/topic'.
   - MensajeConsumerService.java: Escucha los mensajes de Kafka y los despacha al cliente web correspondiente a través del canal '/topic/mensajes/{receptorId}'.
   - MensajeController.java:
     * [HTTP POST] Al crear un mensaje (POST /mensajes), primero se guarda en BBDD y luego se envía a Kafka.
     * [WEBSOCKET] Añadido @MessageMapping("/chat/{receptorId}"). Cuando Angular envía a `/app/chat/{id}`, se guarda en BBDD y se envía a Kafka.

LO QUE DEBES HACER TÚ AHORA (PASO A PASO)
========================================================================

PASO 1: LEVANTAR KAFKA EN TU MÁQUINA
------------------------------------
Asegúrate de tener Docker abierto. Abre una terminal en la raíz de tu proyecto (donde está el archivo docker-compose.yml) y ejecuta:

docker-compose up -d

Esto descargará (la primera vez) y ejecutará un servidor Kafka local.


PASO 2: IMPLEMENTAR EN EL FRONTEND (Angular)
--------------------------------------------
Para que tu aplicación Angular reciba los mensajes en tiempo real, debes seguir estos pasos:

2.1 Instalar dependencias en Angular:
Abre una terminal en la carpeta de tu proyecto Angular y ejecuta:

npm install @stomp/stompjs sockjs-client
npm install -D @types/sockjs-client

2.2 Crear el Servicio WebSocket en Angular:
Crea un servicio (ej: websocket.service.ts) con esta estructura:

import { Injectable } from '@angular/core';
import { Client } from '@stomp/stompjs';
import * as SockJS from 'sockjs-client';
import { Subject } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class WebSocketService {
  private stompClient!: Client;
  private mensajeSubject = new Subject<any>();
  public mensajeObservable = this.mensajeSubject.asObservable();

  constructor() {}

  conectar(miUsuarioId: number) {
    this.stompClient = new Client({
      webSocketFactory: () => new SockJS('http://localhost:8080/api/ws-chat'),
      debug: (str) => console.log(str),
      reconnectDelay: 5000,
    });

    this.stompClient.onConnect = () => {
      console.log('Conectado a WebSocket STOMP');

      // Suscribirse al canal PERSONAL para RECIBIR mensajes
      // (Backend lo envía a /topic/mensajes/{receptorId})
      this.stompClient.subscribe(`/topic/mensajes/${miUsuarioId}`, (mensaje) => {
        if (mensaje.body) {
          this.mensajeSubject.next(JSON.parse(mensaje.body));
        }
      });
    };

    this.stompClient.onStompError = (frame) => {
      console.error('Broker reported error: ' + frame.headers['message']);
      console.error('Additional details: ' + frame.body);
    };

    this.stompClient.activate();
  }

  // MÉTODO PARA ENVIAR MENSAJES VÍA WEBSOCKET
  enviarMensaje(receptorId: number, emisorId: number, contenido: string) {
    if (this.stompClient && this.stompClient.connected) {
      const mensaje = {
        contenido: contenido,
        receptorId: receptorId,
        emisorId: emisorId
      };

      // Enviar al destino mapeado en MensajeController (@MessageMapping("/chat/{receptorId}"))
      // Como configuramos el prefijo "/app" en WebSocketConfig, el destino completo es "/app/chat/{receptorId}"
      this.stompClient.publish({
        destination: `/app/chat/${receptorId}`,
        body: JSON.stringify(mensaje)
      });
    } else {
      console.error('No se pudo enviar el mensaje: Cliente STOMP desconectado.');
    }
  }

  desconectar() {
    if (this.stompClient) {
      this.stompClient.deactivate();
      console.log('Desconectado de WebSocket STOMP');
    }
  }
}
